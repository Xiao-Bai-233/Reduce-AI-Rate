# Reduce AI Rate (论文降AI率)

## Overview

流程化的中文论文降AI率方案。核心认知：**商业AI检测器（知网/维普/PaperYY）抓的不是"AI概率"，而是"AI写作特征"——句长均匀、排比密集、连接词模式化、缺乏个人观点。消除这些特征比拟合检测模型更有效。**

## 总流程

```
.docx路径 → 收集用户需求(风格/阈值/输出设置)
         → python-docx提取纯文本
         → 分类段落(keep vs rewrite)
         → 本人改写(按选定策略逐段重写)
         → 生成输出文件(.docx / .md / .txt)
         → 自检未达标则迭代(最多3轮)
         → 报告结果 + 建议学校工具终检
```

---

## Step 0: 用户信息收集

逐项询问（已有信息跳过，不问重复问题）：

**0.1 论文路径** → 验证文件存在，仅支持 .docx

**0.2 风格选择**
```
1. 专业版 — 保留学术术语和正式语气（学术感 ★★★★★）
2. 平衡版 — 适度"我们"视角，过渡自然（学术感 ★★★★）
3. 口语化版 — 大量口语，短句为主（学术感 ★★★）
```

**0.3 当前AI率**
- 用户知道 → 直接输入百分比（如 65）
- 用户不知道 → 你自己**读几段原文**，凭判断估算AI率
  - 句长是否均匀？"本文/因此/综上所述"这类词是否密集？句式是否模板化？
  - 估算范围即可：≥30%（较高）或 <30%（不算高）
  - 告知用户这是定性估算，建议最终以学校工具为准

AI率 ≥ 30% → 建议**大改**（整体重写段落）
AI率 < 30% → 建议**小改**（局部调整）

询问用户是否同意强度建议，不同意则让用户手动选择大改/小改。

**0.4 目标降AI阈值**（默认 25%）

**0.5 输出文件选择**：docx修改稿 / Markdown对照稿 / 纯文本（默认全生成）

**0.6 保留段落**：用户指定的章节/内容完全跳过，不做任何修改

---

## Step 1: 提取纯文本

用 python-docx 读取文档，保留每个段落的索引、样式名、文本内容。这是纯工具操作——python-docx只用来读和写文件，不做改写逻辑。

```python
from docx import Document
doc = Document(path)
# for i, para in enumerate(doc.paragraphs): 获取 para.text, para.style.name
```

---

## Step 2: 分类段落

**保留（不改写）：**
- 封面信息（学号、姓名、学院、专业、指导教师、日期）
- 诚信声明、知识产权声明
- 目录
- 图题（图1.xxx、表1.xxx）
- 参考文献列表
- 英文摘要与关键词
- 代码块标记行
- **用户指定的保留段落**

**改写：** 其余全部正文（中文摘要、引言、各章节、总结、致谢）

---

## Step 3: 改写（核心步骤）

**执行原则：你亲自来改写。** 逐段阅读原文，理解段落的核心信息点，然后用自己的语言能力重写。不要写脚本/正则来做替换，那是行不通的——语义改写需要理解内容而非匹配模式。

首先确定改写强度，再按选定风格执行：

| | 专业版 | 平衡版 | 口语化版 |
|--|--------|--------|---------|
| **大改** | 深度重构 + 学术语气 | 深度重构 + 适度"我们"视角 | 彻底重构 + 口语表达 |
| **小改** | 局部微调 + 学术语气 | 局部微调 + 自然过渡 | 局部微调 + 少量口语 |

---

### 3.1 大改方向（AI率 ≥ 30%）

大改 = **整体重写**。不以原句为基础做替换，而是理解段落核心信息后，从零组织语言。

**准则：**
- 读完整段，提取论点、论据、结论、数据
- 完全抛开原文句式
- 改变叙事角度、因果顺序、逻辑展开方式
- 确保学术信息完整，但用词、句式、结构不同于原文

**可用的改写手法（灵活选用，不需要全用）：**
- 换叙事角度：原文从"本研究"出发 → 从"问题本身"出发
- 换因果方向：原文"A导致B" → "B的成因是A"
- 换展开方式：原文"先定义后举例" → "先举案例再归纳"
- 调整信息密度：原文密集陈述 → 拆成"观点+展开"两句
- 长句加"呼吸点"：超过40字的长句中间加逗号或拆成两句

**你自己判断的准则：**
- 检查改写后的句子是否还保留了原文的主干结构 → 是的话说明改得不够
- 检查是否还有"本文/本研究/因此/综上所述"等模板词 → 还有的话要换掉
- 检查句长是否均匀 → 均匀的话加入一些短句

**示例（口语化版大改）：**
```
原文：图像分割是计算机视觉领域的重要基础任务。目前主流方法主要分为基于阈值的方法和基于深度学习的方法两类。

改写后：做计算机视觉，图像分割基本是绕不开的。现在主要有两种搞法。一种是老派的阈值分割，速度快是快，但精度不太行。另一种是深度学习，这几年大家都在用，效果确实好很多。
```

---

### 3.2 风格微调方向

选定强度后，按用户风格做语气调整：

**专业版：** 保留术语和正式语气。不加口语表达，不加"我/我们"视角。

**平衡版：** 适度加入"我们"视角。"首先/其次/最后"改成自然过渡（或去掉），"综上所述"改成"总的来看"。保留大部分学术语气。

**口语化版：**
- "本文/本研究" → "我/我们"
- "因此" → "所以说" / "这样一来"
- "然而" → "不过" / "但说实话"
- 长句拆短（每句<40字）
- 段落结构打散，不保留总-分-总
- 减少四字成语和学术词组
- 可以加入"说白了""我觉得""其实就是"

---

### 3.3 小改方向（AI率 < 30%）

小改 = 保留原文骨架，做词汇级 + 轻度句式调整，不改段落结构。

**词汇方向：**
- "本文" → "本研究"
- "目前" → "现阶段" / 去掉
- "随着" → 直接陈述
- "因此" → "基于此" / "鉴于此"

**句式方向：**
- 排比句拆解："首先/其次/最后"结构拆成2-3句，每句换主语或角度
- 段落首句多样化：不都用"XX是YY的重要ZZ"开场
- 句长方差拉大：连续长句之间插入简短评论句
- 连接词替换

**小改不做：** 不拆段落，不重构句子主干，不改句序，不插入个人观点。

---

## Step 4: 输出文件

按用户在 Step 0.5 的选择生成文件。

### Markdown 对照稿
用表格展示原文 vs 改写的对应关系（修改部分着色或不着色均可）。

### 纯文本
只含文字内容，供用户粘贴到检测工具。

### .docx 修改稿
用 python-docx 生成：

```python
from docx import Document
doc = Document(original_path)

# rewritten_map: dict[int, str] = {段落索引: 改写后文本}
for i, para in enumerate(doc.paragraphs):
    if i not in rewritten_map:
        continue
    new_text = rewritten_map[i]
    # 取出原格式
    fmt = {}
    for r in para.runs:
        if r.text.strip():
            fmt = {'name': r.font.name, 'size': r.font.size, 'bold': r.font.bold}
            break
    # 替换文本
    for r in list(para.runs):
        r._element.getparent().remove(r._element)
    new_run = para.add_run(new_text)
    if fmt.get('name'): new_run.font.name = fmt['name']
    if fmt.get('size'): new_run.font.size = fmt['size']

doc.save(output_path)
```

注意：图片/公式位置不变，但文字长度变化可能导致分页变动，属正常现象。

---

## Step 5: 质检测

改写完成后做两件事：

**5.1 自己读一遍**。逐段检查：
- 学术信息有没有遗漏或歪曲？
- 风格是否对应用户选择？
- 该大改的段落是否真正重构了句式？
- 该保留的段落是否原样保留了？

把不合格的段落标记出来。

**5.2 统计AI特征变化**。快速过一遍原文和改写文，估算：
- AI模板词（本文/因此/综上所述等）减少了多少
- 平均句长是否下降
- 句长多样性是否提高

合格的阈值你自己把握——重点看那些"读起来还是像AI写的"段落还有多少。

---

## Step 6: 迭代

如果 Step 5 标记的段落太多 → 回到 Step 3 针对标记段落做第二轮改写，手法比第一轮更激进。最多做3轮。

如果3轮后仍有段落不满意 → 告知用户，建议学校工具终检后针对性处理。

---

## Step 7: 报告结果

1. 告知用户选用的风格和强度
2. AI特征变化概览（模板词减少比例、句长变化）
3. 标记段落的占比
4. **建议用户用学校工具（知网AIGC/PaperYY/维普）终检**
5. 询问是否删除此 skill

---

## 输出文件命名

默认输出到与输入文件同一目录：
- `《论文名》降AI修改稿_{风格}.docx`
- `《论文名》降AI对照稿_{风格}.md`
- `《论文名》降AI纯文本_{风格}.txt`
